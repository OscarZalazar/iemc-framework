# Contributing

Thank you for your interest in IEMC Framework.

This repository is currently focused on public research documentation, examples, terminology, and benchmark design. Contributions should remain academic, transparent, and free of proprietary implementation details.

## What We Welcome

- Conceptual improvements to the IEMC framework
- Literature references and research notes
- Synthetic examples
- Evaluation criteria
- Benchmark proposals
- Documentation fixes
- Public use cases

## What We Do Not Accept

- Private code
- API keys or credentials
- `.env` files
- Proprietary algorithms
- Private prompts
- Production scoring formulas
- Internal configurations
- Non-public datasets
- Content that cannot be redistributed

## Contribution Guidelines

1. Keep examples synthetic and clearly labeled.
2. Avoid claims that require private evidence.
3. Prefer transparent reasoning over opaque scoring.
4. Document assumptions and limitations.
5. Respect cultural context and avoid universalizing emotion labels.

## Pull Request Checklist

- The change is public-safe.
- No secrets or internal configuration are included.
- Examples are fictitious or properly licensed.
- New claims include references or are marked as hypotheses.
- The contribution aligns with Apache 2.0 licensing.
